Recommendation System Notes (Beginner-Friendly for Junior Data Scientists)
==========================================================================

Types of Recommendation Systems
-------------------------------

1. Content-Based Filtering
   - Recommends items using features of the items themselves (e.g., ingredients, category, tags).
   - Matches user preferences to similar item profiles.
   - Data Needed: Recipe data, carbon scores, descriptive features.
   - Use Case: Recommending similar recipes based on ingredients a user already liked.

2. Collaborative Filtering
   - Based on the preferences and behavior of similar users.
   - "Users who liked X also liked Y."
   - Data Needed: User-item interaction data (e.g., ratings, clicks, views).
   - Example Dataset: https://www.kaggle.com/datasets/motahareshokri/cookbook-reviews

3. Knowledge-Based Systems
   - Uses domain-specific rules or logic (e.g., dietary restrictions, preferences).
   - Recommends based on explicitly defined constraints or user input.
   - Good for cold start problems (new users or new items).

4. Hybrid Systems
   - Combines content-based, collaborative, and/or knowledge-based methods.
   - More robust and flexible in real-world use.
   - Example: Netflix uses a hybrid approach for personalization.

What Makes a Good Recommendation System?
----------------------------------------
- Relevance: Accurately reflects user interests.
- Novelty: Introduces fresh or less obvious items.
- Serendipity: Unexpected yet delightful discoveries.
- Diversity: Broad enough selection to maximize engagement.

Cold Start Problem
------------------
Occurs when there is little or no user/item data.

- New App Launch → No data at all.
- New Item → No usage history.
- New User → No prior activity to base predictions on.

Solutions include:
- Use of surveys to gather initial preferences.
- Showing trending or popular content.
- Hybrid models that use item metadata or business rules.
